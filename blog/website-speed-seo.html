웹사이트 속도와 SEO: 이미지가 핵심인 이유

2026년 7월 15일 · 8분 읽기

Google은 2018년부터 모바일 검색에서 속도를 순위 요인으로 사용하기 시작했으며, 2021년에는 Core Web Vitals를 공식 순위 요인으로 도입했습니다. 소개합니다.

핵심 웹 바이탈: Google이 가치를 나타내는 지표

Google이 정의한 한 Core Web Vitals는 세 가지 핵심 지표로 구성됩니다.

LCP (Largest Contentful Paint)

많은 웹사이트에서 LCP Element가 이미지입니다.

  • 좋은 LCP: 2.5초 이하
  • 개선 필요: 2.5~4초
  • 나쁨: 4초 이상

이미지를 압축하고 적절한 크기로 제공하면 LCP 시간을 상당히 할 수 있습니다.

FID (First Input Delay) / INP (Interaction to Next Paint)

사용자 입력에 대한 응답 시간입니다. “이외의 질문이 있을 수 있습니다.

CLS (Cumulative Layout Shift)

이미지 크기를 우편으로 보내지 않으면 이미지가 로드될 시간이 갑자기 이동합니다.

이미지가 SEO에 미치는 영향

1. 페이지 로딩 속도

이미지는 평균 웹 페이지의 전송 데이터 52%를 차지합니다. 이미지를 압축하면 전체 페이지의 크기가 대폭으로 늘어나는 속도가 향상됩니다.

2. 이탈률

Google Research에 따르면, 페이지 로드 시간이 1초에서 3초로 인해면 이탈률이 32% 증가합니다. 빠른 하중은 이탈률로 인해, Search Engine에서 긍정적인 신호로 작용합니다.

3. 크롤링 효율

Google의 크롤러는 제한된 크롤링 쥐를 가지고 있습니다. 크롤링할 수 있어, 사이트의 more 콘텐츠가INDEX에 존재합니다.

4. 모바일 우선 색인

Google은 2023년부터 완전히 우선적으로 색상으로 전환했습니다. 모바일 환경의 성능이 직접적인 검색 결과에 영향을 미칩니다.

이미지 SEO 모범 사례

1. 현대 이미지 포맷 사용

WebP나 AVIF 같은 현대 포맷을 사용하여 파일 크기를 줄이세요. ZXZX의 변환 도구를 사용하면 쉽게 변환할 수 있습니다.

2. 적절한 크기로 리사이즈

화면 표시 크기에 맞는 이미지를 제공하세요. 1920px 이미지를 400px 영역에 표시하는 것은 불필요한 낭비입니다.

3. 지연 로딩 (Lazy Loading)

폴딩 아래 이미지에는 loading="lazy" 속성을 추가하세요. 초기 로딩 속도가개선되고 불필요한 리소스 로딩을 방지합니다.

4. alt 텍스트 최적화

이미지 alt는 믿음이 없는 SEO에도 중요합니다. 키워드를 자연스럽게 포함하는 설명을 작성하세요.

5. 반응형 이미지

srcset과 sizes 속성을 사용하여 디바이스에 맞는 크기를 제공하세요.

실제 사례: 이미지 최적화 후 SEO개선

한 블로그 사이트의 이미지를 종합적으로 최적화한 결과입니다:

Before

  • 이미지 총 용량: 4.2MB
  • 포맷: JPG, PNG 혼합
  • LCP: 4.8초
  • CLS: 0.25
  • PageSpeed 점수: 45점
  • 검색 순위: 평균 15위

After (WebP 변환 + 압축 + 리사이즈)

  • 이미지 총 용량: 1.1MB (74% 감소)
  • 포맷: WebP 통일
  • LCP: 1.9초 (60%개선)
  • CLS: 0.05
  • PageSpeed 점수: 89점
  • 검색 순위: 평균 8위 (47%개선)

Google PageSpeed Insights 분석

PageSpeed Insights는 이미지 관련하여 다음과 같은제안을제공합니다:

  • "차세대 형식으로 이미지 제공": WebPna AVIF로 보여주라고 권장합니다
  • "적절한 크기의 이미지": 이미지 크기를 조정하라는 권장사항입니다.
  • "오프스크린 이미지 연기": 폴딩 아래 이미지에 지연 로드를 적용하도록 권장합니다.
  • "이미지를 효율적으로 인코딩": 이미지 압축을 최적화하라는 권장사항입니다

이미지 최적화 체크리스트

  • ☐ 모든 이미지가 WebP 또는 AVIF 포맷으로 변환됨
  • ☐ 품질 수준이 75~85%로 설정됨
  • ☐ 화면 표시 크기에 맞게 리사이즈됨
  • ☐ width, height 속성이 지정됨 (CLS 방지)
  • ☐ 폴딩 아래 이미지에 loading="lazy" 적용됨
  • ☐ srcset으로 반응형 이미지 제공됨
  • ☐ 모든 이미지에 설명적인 alt 텍스트가 포함됨
  • ☐ 이미지가 적절한 압축을 거침

ZXZX로 이미지 SEO 최적화하기

  2. 변환 도구: JPG, PNG를 WebP나 AVIF로 변환합니다
  3. 리사이즈 도구: 적절한 크기로 이미지를 조정합니다
