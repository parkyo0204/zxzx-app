문의하기

질문, 피드백, 또는 파트너십 문의가 있으신가요? 저희 팀은 모든 메시지를 읽으며 보통 1-2 영업일 이내에 답변드립니다.

연락하기

아래 문의 양식을 통해 연락해 주세요. 버그 리포트, 기능 요청, 파트너십 제안, ZXZX에 대한 일반적인 질문 등 모든 유형의 문의를 환영합니다.

문의 후 기대할 수 있는 것

사용자로부터 받는 모든 메시지를 소중히 여낍니다. ZXZX 팀에 연락하실 때 커뮤니케이션 과정에서 기대할 수 있는 것을 소개합니다. 투명성과 신속한 응답을 믿으며, 모든 문의를 철저히 처리하기 위해 노력합니다.

문의 양식을 제출하시면 메시지를 받았음을 확인하는 자동 확인 이메일을 받게 됩니다. 팀은 영업일 동안 매일 수신 메시지를 검토합니다. 일반적인 질문과 피드백은 보통 24시간 이내에 답변됩니다. 기술적 문제와 버그 리포트는 추가 조사 시간이 필요할 수 있지만, 접수를 확인하고 해결 예상 일정을 항상 제공합니다.

파트너십 및 비즈니스 문의의 경우, 팀에서 각 제안을 신중하게 검토합니다. 무료 프라이버시 중심 이미지 도구를 제공한다는 미션과 일치하는 협업에 열려 있습니다. 귀하의 조직과 제안된 파트너십의 성격에 대해 가능한 한 많은 세부 정보를 포함해 주시면 적절한 팀원에게 문의를 전달할 수 있습니다.

도움을 드릴 수 있는 주요 주제

지원팀은 다양한 질문과 문제를 처리할 수 있는 역력을 갖추고 있습니다. 사용자가 가장 자주 문의하는 주요 주제와 메시지에 포함할 정보에 대한 안내를 소개합니다.

  • 도구가 작동하지 않음 — ZXZX 도구가 예상대로 작동하지 않는 경우, 브라우저 이름과 버전, 운영 체제, 사용 중이던 특정 도구, 처리 중이던 이미지의 파일 유형과 크기를 포함해 주세요. 이 정보가 문제를 빠르게 재현하고 수정하는 데 도움이 됩니다.
  • 기능 제안 — 사용자 피드백을 기반으로 새로운 기능의 우선순위를 적극적으로 정하고 있습니다. ZXZX에서 보고 싶은 도구나 기능이 있으면 사용 사례를 설명하고 워크플로우에 어떻게 도움이 되는지 알려주세요. 세부 정보가 많을수록 제안을 더 잘 평가할 수 있습니다.
  • 콘텐츠 오류 — 블로그 기사, 도구 설명, 문서에서 오류를 발견하면 해당 페이지 URL과 부정확한 내용을 알려주세요. 정확하고 최신의 정보를 제공하기 위해 노력하며 오류를 신속하게 수정합니다.
  • 광고 관련 우려 — ZXZX에서 부적절하거나 오해의 소지가 있거나 사용자 경험을 방해하는 광고를 발견하면 가능한 한 자세히 신고해 주세요. 사용자 경험을 긍정적으로 유지하기 위해 광고 파트너와 협력하며 광고 품질을 중요하게 생각합니다.
  • 접근성 문제 — ZXZX는 모든 사용자가 접근할 수 있도록 최선을 다하고 있습니다. 도구 사용이나 웹사이트 탐색 중 접근성 장벽에 직면하면 문제와 사용 중인 보조 기술을 설명해 주시면 해결하기 위해 노력하겠습니다.

다른 연락 방법

이메일

직접 연락하려면 다음으로 이메일을 보내주세요: hello@zxzx.app

모든 이메일에 1-2 영업일 이내에 답변드립니다. 적절한 팀원에게 메시지를 전달할 수 있도록 명확한 제목을 포함해 주세요. 긴급한 사안의 경우 제목에 [URGENT]를 표시해 주시면 문의를 우선 처리하겠습니다.

소셜 미디어

ZXZX 도구와 기능에 대한 업데이트, 팁, 소식을 위해 팔로우해 주세요. 소셜 채널에서 이미지 최적화 팁, 웹 성능 인사이트, 새 도구에 대한 안내를 정기적으로 공유합니다. 소셜 미디어는 플랫폼 업데이트와 유지보수 일정에 대한 정보를 얻는 좋은 방법이기도 합니다.

커뮤니티 및 피드백

ZXZX는 커뮤니티를 위해 만들어졌으며, 여러분의 피드백이 제품 로드맵을 직접 형성합니다. 출시 이후 배치 처리, ZIP 다운로드, AVIF 지원 등 수십 가지 사용자 요청 기능을 구현했습니다. 받는 모든 피드백은 제품 팀에서 검토하고 향후 개발에 반영합니다.

자주 묻는 질문

답변은 얼마나 빨리 받을 수 있나요?

모든 문의에 1-2 영업일 이내에 답변드리기 위해 노력합니다. 복잡한 기술적 문제나 파트너십 제안은 관련 팀원과 상의해야 하므로 다소 시간이 더 걸릴 수 있습니다.

버그를 발견했습니다. 리포트에 무엇을 포함해야 하나요?

버그를 리포트하실 때 다음을 포함해 주세요: (1) 사용 중인 브라우저와 기기, (2) 관련된 특정 도구와 파일 유형, (3) 예상했던 동작, (4) 실제로 발생한 동작. 가능한 경우 스크린샷이 매우 도움이 됩니다.

새 기능을 요청할 수 있나요?

물론이죠! 어떤 기능이 가장 유용할지 사용자 여러분의 의견을 듣고 싶습니다. 문의 양식에서 주제로 "기능 요청"을 선택해 주세요. 모든 제안을 구현할 수는 없지만, 사용자 수요와 기술적 타당성을 기반으로 기능 우선순위를 정합니다.

API 접근이나 엔터프라이즈 솔루션을 제공하나요?

현재 ZXZX는 공개 API가 없는 브라우저 기반 도구입니다. API 접근이나 엔터프라이즈 솔루션에 관심이 있으시면 사용 사례에 대한 세부 정보와 함께 문의 양식으로 연락해 주세요.

프라이버시 우려 사항은 어떻게 보고하나요?

프라이버시 우려 사항을 심각하게 받아들�니다. 문의 양식 주제 드롭다운에서 "프라이버시 관련"을 선택하시면 프라이버시 팀이 조사하고 신속하게 답변드리겠습니다. ZXZX의 데이터 처리 방식에 대한 자세한 내용은 개인정보 처리방침을 참조하세요.
